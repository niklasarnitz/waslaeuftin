## 2024-05-10 - CommandSearch accessibility mapping (Combobox)
**Learning:** When implementing custom search dialogs/menus using an `input` combined with a dynamic results list, simply adding an `input` is not enough for screen readers. The `combobox` ARIA pattern must be fully implemented to link the input with the listbox using `aria-controls` and `aria-activedescendant`. Syncing custom arrow-key state with native Tab navigation by attaching `onFocus` handlers to the options ensures visual focus remains synchronized. Furthermore, when adding a clear button to an input, using `useRef` rather than `document.querySelector` is crucial to prevent focus jumps that break the dialog's keyboard navigation.
**Action:** Always fully implement the WAI-ARIA combobox pattern (`role="combobox"`, `role="listbox"`, `role="option"`, `aria-activedescendant`) for custom auto-complete widgets, and always rely on React `useRef` for focus management, avoiding raw DOM selectors that could target inputs outside the current component's scope.
